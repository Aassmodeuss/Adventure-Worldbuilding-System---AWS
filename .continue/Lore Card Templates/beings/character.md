# Template - Being: Character

Purpose
- Use for individual NPCs/PCs.
- Detail role, appearance cues, behavior patterns, and costs/limits.

Frontmatter (optional parent)
```yaml
---
id: character:lethra-vinecaller
type: character
name: Lethra Vinecaller
parent: location:westgrove     # optional, use if domiciled clearly
see_also: []
---
```

Card template
```markdown
[ENTRY: {Name}]

#### [Character]
**Name:**  
**Aliases:**  
**Summary:** 1-2 sentences on identity and significance. Write this as a standalone section; do not rely on Full Description.  
**Full Description:** 4-10 sentences on history, appearance cues, hallmark behaviors, traits, abilities, and limits. Write this as a standalone section; do not rely on Summary.  
**Tone Keywords:** 3-5 anchors.  
**Race:** exact race card name(s)  
**Class/Role:** exact role/class card name(s)  
**Factions:** comma-separated faction card names (membership or command)  
**Known Associates:** comma-separated close ties (character name - relation)  
**Potential Gameplay Hooks:** 1-3 concise bullets describing potential interactions, quests, or story events involving the character.


[END: {Name}]
```



Pre-save checklist
- Race, Class/Role, and Factions fields populated.  
- Known Associates uses "Name - relation" format for clarity.  
- Parent set if the character is domiciled clearly.
